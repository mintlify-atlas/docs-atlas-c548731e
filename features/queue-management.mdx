## Overview

The Downloads page provides a comprehensive view of all your active, completed, and failed downloads. Track real-time progress, manage queues, and access your download history.

## Download Queue Interface

## Progress Events

Subscribe to real-time download updates:

```tsx
// Listen for yt-dlp progress
useEffect(() => {
  const unlisten = api.onDownloadProgress((progress) => {
    setProgressMap(prev => {
      const newMap = new Map(prev);
      const stabilized = stabilizeProgress(progress);
      newMap.set(progress.id, stabilized);
      return newMap;
    });
    
    // Update status in database when complete
    if (progress.status === 'completed' || 
        progress.status === 'failed') {
      api.updateDownloadStatus(progress.id, progress.status);
      setTimeout(loadDownloads, 500);
    }
  });
  
  return () => unlisten();
}, []);
```

**Progress Stabilization:**

```typescript
// Prevent progress from jumping backwards
const stabilizeProgress = (
  incoming: DownloadProgress, 
  previous?: DownloadProgress
): DownloadProgress => {
  const normalized = Math.min(100, Math.max(0, incoming.progress));
  
  if (!previous) {
    return { ...incoming, progress: normalized };
  }
  
  // Keep progress monotonic during active phases
  if (incoming.status === 'downloading') {
    return {
      ...incoming,
      progress: Math.max(previous.progress, normalized)
    };
  }
  
  return { ...incoming, progress: normalized };
};
```

## Download Actions

### Cancel Download

Stop an active download:

```tsx
const handleCancel = async (id: string) => {
  try {
    // Try yt-dlp cancel first
    try {
      await api.cancelDownload(id);
    } catch {
      // Fallback to Spotify cancel
      await api.cancelSpotifyDownload(id);
    }
    
    toast.success('Download cancelled');
    await api.updateDownloadStatus(id, 'cancelled');
    loadDownloads();
  } catch (err) {
    toast.error('Failed to cancel download');
  }
};
```

**Backend cancellation:**

```rust
// Cancel signal handling
let (cancel_tx, mut cancel_rx) = tokio::sync::oneshot::channel();

// Store cancellation sender
ACTIVE_DOWNLOADS.lock().unwrap()
  .insert(download_id.clone(), cancel_tx);

// Download loop with cancellation
tokio::select! {
  _ = &mut cancel_rx => {
    // User cancelled
    let _ = child.kill().await;
    emit_progress("cancelled");
  }
  result = child.wait() => {
    // Download completed
  }
}
```

### Delete from History

Remove a download record:

```tsx
const handleDelete = async (id: string) => {
  try {
    await api.deleteDownload(id);
    setDownloads(prev => prev.filter(d => d.id !== id));
    toast.success('Download removed');
  } catch (err) {
    toast.error('Failed to remove download');
  }
};
```

### Clear All Downloads

```tsx
const handleClearAll = async () => {
  try {
    await api.clearDownloads();
    setDownloads([]);
    toast.success('All downloads cleared');
  } catch (err) {
    toast.error('Failed to clear downloads');
  }
};
```

### History Display

```tsx
// Format time ago
const formatTimeAgo = (timestamp: number): string => {
  const diff = Date.now() - timestamp;
  const seconds = Math.floor(diff / 1000);
  const minutes = Math.floor(seconds / 60);
  const hours = Math.floor(minutes / 60);
  const days = Math.floor(hours / 24);
  
  if (seconds < 60) return 'Just now';
  if (minutes < 60) return `${minutes}m ago`;
  if (hours < 24) return `${hours}h ago`;
  if (days < 7) return `${days}d ago`;
  
  return new Date(timestamp).toLocaleDateString();
};

// History card
<div className="history-item">
  <img src={item.thumbnail} alt={item.title} />
  <div>
    <h3>{item.title}</h3>
    <p>{item.platform} • {formatBytes(item.size_bytes)}</p>
    <span>{formatTimeAgo(item.timestamp)}</span>
  </div>
</div>
```

## Playback Integration

Play downloaded files directly from the Downloads page:

```tsx
const handlePlay = async (path: string, title: string) => {
  try {
    const mediaInfo = await api.findMediaFile(path, title);
    let playablePath = mediaInfo.file_path;
    
    if (!mediaInfo.is_audio) {
      // Transcode video for web playback
      const result = await api.transcodeForPlayback(playablePath);
      playablePath = result.output_path;
    }
    
    setPlayerFilePath(playablePath);
    setPlayerTitle(title);
    setPlayerIsAudio(mediaInfo.is_audio);
    setShowPlayer(true);
  } catch (err) {
    toast.error('Failed to find media file');
  }
};
```

## Database Schema

Downloads are stored in SQLite:

```sql
CREATE TABLE downloads (
    id TEXT PRIMARY KEY,
    url TEXT NOT NULL,
    title TEXT NOT NULL,
    platform TEXT,
    format TEXT,
    status TEXT NOT NULL,  -- 'pending', 'downloading', 'completed', 'failed'
    path TEXT,
    size_bytes INTEGER,
    thumbnail TEXT,
    timestamp INTEGER NOT NULL,
    metadata TEXT  -- JSON for additional data
);

CREATE INDEX idx_downloads_timestamp ON downloads(timestamp DESC);
CREATE INDEX idx_downloads_status ON downloads(status);
```

## Best Practices

<AccordionGroup>
  <Accordion title="Concurrent Downloads">

    ```tsx
    const MAX_CONCURRENT = 3;
    ```

    ```typescript
    // Check available space
    const checkDiskSpace = async (requiredBytes: number) => {
      const stats = await api.getDiskSpace();

        throw new Error('Insufficient disk space');
      }
    ```

  </Accordion>

## Troubleshooting

### Progress Not Updating

1. Check WebSocket connection
2. Verify download process is active
3. Restart the app if progress is frozen

### Download Stuck at 99%

This usually means yt-dlp is merging video/audio streams:

```rust
// Post-processing phase
if line.contains("[merger]") || line.contains("merging formats") {
    emit_progress(DownloadProgress {
        progress: 99.0,
        speed: "Merging...",
        status: "downloading",
    });
}
```
