The **Smart Network Download Engine (SNDE)** is Ownstash Downloader's high-performance parallel download system designed to maximize bandwidth utilization for static files.

## Overview

SNDE accelerates downloads by splitting files into chunks and downloading them simultaneously over multiple TCP connections, similar to how IDM (Internet Download Manager) works.

**Location**: `src-tauri/src/snde.rs`

## How It Works

### 1. Preflight Probe

Before downloading, SNDE probes the server to determine capabilities:

```rust
async fn probe_file(&self, request: &SNDERequest) 
    -> Result<(u64, bool, Option<String>), String> 
{
    let response = self.client
        .head(&request.url)
        .header(USER_AGENT, "Mozilla/5.0...")
        .send()
        .await?;

    let content_length = headers.get(CONTENT_LENGTH)?;
    let supports_range = headers.get(ACCEPT_RANGES) == "bytes";
    
    Ok((content_length, supports_range, filename))
}
```

**Checks:**
- File size (`Content-Length` header)
- Range request support (`Accept-Ranges: bytes`)
- Filename from `Content-Disposition` header
- HTTP version negotiation

### 2. File Pre-Allocation

SNDE pre-allocates disk space to optimize write performance:

```rust
async fn preallocate_file(&self, path: &PathBuf, size: u64) -> Result<(), String> {
    let file = File::create(path).await?;
    
    if size < 8 * 1024 * 1024 * 1024 {
        // < 8GB: Immediate full pre-allocation
        file.set_len(size).await?;
    } else if size < 32 * 1024 * 1024 * 1024 {
        // 8-32GB: Chunked pre-allocation (first 4GB)
        file.set_len(4 * 1024 * 1024 * 1024).await?;
    } else {
        // > 32GB: Lazy growth (allocate on demand)
    }
    
    Ok(())
}
```

<Info>
  Pre-allocation reduces file fragmentation and improves sequential write performance on NTFS/ext4.
</Info>

### 3. Chunk Distribution

The file is divided into work units for parallel processing:

```rust
fn create_chunks(&self, total_size: u64, num_connections: u8) -> Vec<ChunkWork> {
    let chunk_size = (total_size / num_connections as u64)
        .max(MIN_CHUNK_SIZE); // Minimum 1MB chunks
    
    let mut chunks = Vec::new();
    let mut start = 0u64;
    
    while start < total_size {
        let end = (start + chunk_size - 1).min(total_size - 1);
        chunks.push(ChunkWork {
            start,
            end,
            in_progress: false,
            completed: false,
            retries: 0,
        });
        start = end + 1;
    }
    
    chunks
}
```

### 4. Worker Threads

Each connection runs as an async worker that claims and processes chunks:

```rust
async fn worker_loop(
    conn_id: u8,
    client: Client,
    url: String,
    chunks: Arc<Mutex<Vec<ChunkWork>>>,
    file: Arc<Mutex<File>>,
    total_downloaded: Arc<AtomicU64>,
    is_cancelled: Arc<AtomicBool>,
) -> bool {
    loop {
        // 1. Claim next available chunk
        let (start, end, chunk_idx) = claim_chunk(&chunks)?;
        
        // 2. Download chunk with Range request
        let success = download_chunk(
            &client, &url, start, end, 
            file.clone(), total_downloaded.clone()
        ).await;
        
        // 3. Mark chunk as completed or retry
        update_chunk_status(&chunks, chunk_idx, success);
    }
}
```

### 5. Range Request Download

Each chunk is downloaded using HTTP Range requests:

```rust
async fn download_chunk(
    client: &Client,
    url: &str,
    start: u64,
    end: u64,
    file: Arc<Mutex<File>>,
    total_downloaded: Arc<AtomicU64>,
) -> bool {
    let range_header = format!("bytes={}-{}", start, end);
    
    let response = client
        .get(url)
        .header(RANGE, &range_header)
        .send()
        .await?;
    
    let mut stream = response.bytes_stream();
    let mut position = start;
    
    while let Some(chunk) = stream.next().await {
        let bytes = chunk?;
        
        // Write to file at correct position
        let mut file_guard = file.lock().await;
        file_guard.seek(SeekFrom::Start(position)).await?;
        file_guard.write_all(&bytes).await?;
        
        position += bytes.len() as u64;
        total_downloaded.fetch_add(bytes.len() as u64, Ordering::Relaxed);
    }
    
    true
}
```

## Configuration

### Constants

```rust
/// Maximum concurrent connections per download
const MAX_CONNECTIONS: u8 = 16;

/// Minimum chunk size (1MB) - don't split below this
const MIN_CHUNK_SIZE: u64 = 1024 * 1024;

/// Default chunk size for work distribution (8MB)
const DEFAULT_CHUNK_SIZE: u64 = 8 * 1024 * 1024;

/// Buffer size for reading response body (256KB)
const BUFFER_SIZE: usize = 256 * 1024;

/// Stall detection timeout (10 seconds with no progress)
const STALL_TIMEOUT_SECS: u64 = 10;

/// Throttling threshold - speed drops below 30% of peak
const THROTTLE_THRESHOLD_PERCENT: f64 = 0.3;
```

### Dynamic Connection Scaling

The number of connections is determined by:

1. **Host Reputation** (if available):
   ```rust
   let recommended_connections = host_reputation.max_stable_conns;
   ```

2. **File Size** (for new hosts):
   ```rust
   let recommended_connections = match file_size {
       size if size > 100_000_000 => 8,  // >100MB: 8 connections
       size if size > 10_000_000  => 6,  // >10MB:  6 connections  
       size if size > 1_000_000   => 4,  // >1MB:   4 connections
       _                          => 2,  // Small:  2 connections
   };
   ```

3. **Range Support**:
   ```rust
   let num_connections = if supports_range {
       recommended_connections.min(MAX_CONNECTIONS)
   } else {
       1  // Fallback to single connection
   };
   ```

## Progress Reporting

SNDE emits real-time progress events to the frontend:

```rust
let progress_handle = tokio::spawn(async move {
    let mut last_bytes = 0u64;
    let mut last_time = Instant::now();
    
    while !is_cancelled.load(Ordering::Relaxed) {
        tokio::time::sleep(Duration::from_millis(250)).await;
        
        let current_bytes = total_downloaded.load(Ordering::Relaxed);
        let elapsed = last_time.elapsed().as_secs_f64();
        
        // Calculate speed
        let speed_bps = ((current_bytes - last_bytes) as f64 / elapsed) as u64;
        let progress = (current_bytes as f64 / total_size as f64) * 100.0;
        
        // Calculate ETA
        let remaining_bytes = total_size.saturating_sub(current_bytes);
        let eta_secs = if speed_bps > 0 {
            remaining_bytes / speed_bps
        } else {
            0
        };
        
        app.emit("download-progress", SNDEProgress {
            id: id.clone(),
            progress,
            speed: format_speed(speed_bps),
            eta: format_eta(eta_secs),
            downloaded_bytes: current_bytes as i64,
            total_bytes: total_size as i64,
            active_connections: num_connections,
            engine_badge: "SNDE ACCELERATED".to_string(),
        });
        
        last_bytes = current_bytes;
        last_time = Instant::now();
    }
});
```

### Event Structure

```typescript
interface SNDEProgress {
  id: string;                    // Download ID
  progress: number;              // 0-100 percentage
  speed: string;                 // "5.2 MB/s"
  eta: string;                   // "2m 30s" or "1h 5m"
  status: string;                // "downloading" | "completed" | "failed"
  downloaded_bytes: number;      // Bytes downloaded so far
  total_bytes: number;           // Total file size
  active_connections: number;    // Current connection count
  engine_badge: string;          // "SNDE ACCELERATED" | "SNDE SAFE"
}
```

## HTTP/1.1 vs HTTP/2

### Why Force HTTP/1.1?

SNDE uses a dedicated HTTP/1.1-only client for parallel downloads:

```rust
let http1_client = Client::builder()
    .timeout(Duration::from_secs(300))
    .connect_timeout(Duration::from_secs(30))
    .pool_max_idle_per_host(MAX_CONNECTIONS as usize)
    .http1_only()  // Force HTTP/1.1
    .build()
    .unwrap();
```

**Reason**: HTTP/2 multiplexes all streams over a single TCP connection, which defeats the purpose of parallel connections. HTTP/1.1 creates separate TCP connections, allowing true parallelism.

<Note>
  **HTTP/1.1**: Each connection = separate TCP socket = parallel bandwidth  
  **HTTP/2**: All streams = single TCP socket = multiplexed (no parallel gain)
</Note>

### Protocol Selection

The Download Router decides which client to use:

```rust
pub struct RoutingDecision {
    pub engine: DownloadEngine,
    pub force_http1: bool,  // Use HTTP/1.1 client?
    // ...
}

if routing_decision.force_http1 {
    self.http1_client.clone()
} else {
    self.client.clone()
}
```

## Error Handling & Retries

### Chunk Retry Logic

```rust
struct ChunkWork {
    start: u64,
    end: u64,
    in_progress: bool,
    completed: bool,
    retries: u8,  // Retry counter
}

// Worker loop checks retry count
for (idx, c) in chunks_guard.iter_mut().enumerate() {
    if !c.completed && !c.in_progress && c.retries < 5 {
        c.in_progress = true;
        found = Some((c.start, c.end, idx));
        break;
    }
}

// After failed chunk
if !result {
    chunk.retries += 1;
    println!("[SNDE] Worker {} failed chunk {}-{}, retry {}", 
             conn_id, start, end, chunk.retries);
}
```

<Warning>
  After 5 failed retries, the chunk is abandoned and the download fails.
</Warning>

### Connection Failure Recovery

If a worker panics or fails:

```rust
let mut all_success = true;
for handle in worker_handles {
    match handle.await {
        Ok(result) => {
            if !result {
                all_success = false;
            }
        }
        Err(e) => {
            println!("[SNDE] Worker panicked: {:?}", e);
            all_success = false;
        }
    }
}
```

Other workers continue processing remaining chunks.

## Integration with Host Reputation

SNDE learns from each download and updates the Host Reputation database:

```rust
// Before download: get optimal settings
let host_reputation = reputation_manager.get_reputation(&domain)?;
let recommended_connections = host_reputation.max_stable_conns;

// After download: update reputation
reputation_manager.update_after_download(
    &domain,
    DownloadStats {
        success: true,
        avg_speed_kbps: 5120,
        num_connections_used: 8,
        throttled: false,
    }
)?;
```

### Reputation-Based Decisions

```rust
let (engine, badge) = if let Some(ref rep) = host_reputation {
    if rep.health_score < 30 || rep.max_stable_conns <= 1 {
        // Poor reputation - use safe mode
        (DownloadEngine::SNDESafe, "SNDE SAFE".to_string())
    } else {
        // Good reputation - use accelerated mode
        (DownloadEngine::SNDE, "SNDE ACCELERATED".to_string())
    }
} else {
    // New host - default to accelerated
    (DownloadEngine::SNDE, "SNDE ACCELERATED".to_string())
};
```

### Benchmark Results

<Info>
  SNDE can achieve **3-8x speed improvements** on high-bandwidth connections when downloading from servers that support Range requests and don't throttle.
</Info>

**Example: 100MB file download**

| Connections | Time    | Speed     | Improvement |
|-------------|---------|-----------|-------------|
| 1           | 160s    | 5.0 MB/s  | Baseline    |
| 4           | 50s     | 16.0 MB/s | 3.2x        |
| 8           | 28s     | 28.6 MB/s | 5.7x        |
| 16          | 25s     | 32.0 MB/s | 6.4x        |

<Note>
  Actual performance depends on network bandwidth, server capabilities, and disk write speed.
</Note>

### Bottleneck Analysis

1. **Network Bandwidth**: Maximum theoretical speed
2. **Server Rate Limiting**: Many CDNs limit per-connection speed
3. **Disk I/O**: Write speed to storage device
4. **CPU**: Minimal impact (async I/O)

## Troubleshooting

### Issue: Downloads slower than single connection

**Cause**: Server may be throttling parallel connections.

**Solution**: SNDE automatically detects throttling and reduces connections. Check Host Reputation:

```rust
let reputation = reputation_manager.get_reputation(&domain)?;
println!("Max stable connections: {}", reputation.max_stable_conns);
println!("Health score: {}", reputation.health_score);
```

### Issue: Download fails with "Range not supported"

**Cause**: Server doesn't support Range requests.

**Solution**: Download Router automatically falls back to `SNDESafe` (single connection) or Media Engine.

### Issue: File corruption after download

**Cause**: Chunk write conflicts or incomplete chunks.

**Solution**: Each chunk has retry logic. Check logs for failed chunks:

```
[SNDE] Worker 3 failed chunk 8388608-16777215, retry 1
[SNDE] Worker 3 failed chunk 8388608-16777215, retry 2
```
