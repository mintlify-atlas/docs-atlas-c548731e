## Prerequisites

Before you begin, ensure you have the following tools installed:

### Required Tools

<Steps>
  <Step title="Node.js 20+">
    Download from [nodejs.org](https://nodejs.org/) or use a version manager:
    
    ```bash
    # Verify installation
    node --version  # Should be v20.0.0 or higher
    npm --version
    ```
  </Step>
  
  <Step title="Rust Stable Toolchain">
    Install via [rustup](https://rustup.rs/):
    
    ```bash
    # Install rustup (Windows)
    winget install Rustlang.Rustup
    
    # Or download from rustup.rs
    curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
    
    # Verify installation
    rustc --version
    cargo --version
    ```
  </Step>
  
  <Step title="Windows PowerShell" icon="windows">
    Required for the binary bootstrap script. Comes pre-installed on Windows.
    
    ```powershell
    # Verify PowerShell version
    $PSVersionTable.PSVersion  # Should be 5.1 or higher
    ```
  </Step>
</Steps>

## Cloning the Repository

Clone the repository from GitHub:

```bash
git clone https://github.com/SlasshyOverhere/Ownstash-Downloader.git
cd Ownstash-Downloader
```

## Development Setup

### Step 1: Install Dependencies

Install Node.js dependencies:

```bash
npm install
```

This installs all frontend dependencies listed in `package.json`:

- React 19 + TypeScript
- Vite build tooling
- TailwindCSS + PostCSS
- Tauri CLI
- UI libraries (Radix UI, Framer Motion, etc.)

### Step 2: Download Bundled Binaries

Download the required external tools (yt-dlp, SpotDL, FFmpeg):

```bash
npm run download-binaries
```

This runs the PowerShell script `src-tauri/download-binaries.ps1`:

<CodeGroup>
```powershell src-tauri/download-binaries.ps1
# Creates binaries directory
$binariesDir = Join-Path $PSScriptRoot "binaries"
New-Item -ItemType Directory -Force -Path $binariesDir

# Downloads:
# - yt-dlp.exe from yt-dlp/yt-dlp releases
# - spotdl.exe from spotDL/spotify-downloader releases  
# - ffmpeg.exe and ffprobe.exe from FFmpeg-Builds

Write-Host "Downloading yt-dlp..." -ForegroundColor Cyan
$ytdlpUrl = "https://github.com/yt-dlp/yt-dlp/releases/latest/download/yt-dlp.exe"
Invoke-WebRequest -Uri $ytdlpUrl -OutFile "$binariesDir/yt-dlp.exe"

# ... (similar for SpotDL and FFmpeg)
```
</CodeGroup>

**Downloaded Files:**
- `src-tauri/binaries/yt-dlp.exe` - Media downloader
- `src-tauri/binaries/spotdl.exe` - Spotify downloader
- `src-tauri/binaries/ffmpeg.exe` - Media encoder
- `src-tauri/binaries/ffprobe.exe` - Media analyzer

<Warning>
  These binaries are **bundled into the installer** but are **not committed to git**. You must run this script before building.
</Warning>

### Step 3: Start Development Server

Run the development server with hot reload:

```bash
npm run tauri dev
```

This command:

1. Starts Vite dev server on `http://localhost:1420`
2. Compiles Rust backend in debug mode
3. Launches the app with hot reload enabled

**What happens:**
- Frontend changes trigger instant HMR (Hot Module Replacement)
- Rust changes trigger automatic recompilation and app restart
- Console logs from both frontend and backend appear in terminal

<Info>
  **First run** may take 2-5 minutes to compile Rust dependencies. Subsequent runs are much faster due to caching.
</Info>

### Development Workflow

```bash
# Terminal 1: Run dev server
npm run tauri dev

# Terminal 2: Run frontend separately (optional)
npm run dev

# Terminal 3: Build frontend only
npm run build

# Check TypeScript types
npx tsc --noEmit

# Lint code
npm run lint
```

## Production Build

### Full Installer Build

Build the complete installer with all dependencies:

```bash
npm run build-installer
```

This is equivalent to:

```bash
npm run download-binaries && npm run tauri build
```

**Build Steps:**

<Steps>
  <Step title="Download Binaries">
    Runs `download-binaries.ps1` to fetch external tools
  </Step>
  
  <Step title="Build Frontend">
    Vite builds the React app:
    - TypeScript compilation
    - Code minification
    - Asset optimization
    - Output to `dist/`
  </Step>
  
  <Step title="Build Backend">
    Cargo builds Rust code in release mode:
    - Optimizations enabled (`opt-level = "s"`)
    - Link-time optimization (LTO)
    - Binary stripping
    - Output to `src-tauri/target/release/`
  </Step>
  
  <Step title="Create Installer">
    Tauri bundles everything:
    - Embeds WebView2
    - Bundles external binaries
    - Creates NSIS installer (Windows)
    - Generates updater artifacts
  </Step>
</Steps>

### Build Outputs

After building, find the artifacts in:

```
src-tauri/target/release/bundle/
├── nsis/
│   └── ownstash-downloader-windows-x64-setup.exe  # Main installer
├── msi/
│   └── ownstash-downloader-windows-x64.msi        # MSI installer
└── updater/
    ├── ownstash-downloader-windows-x64-setup.nsis.zip      # Updater archive
    └── ownstash-downloader-windows-x64-setup.nsis.zip.sig  # Signature
```

**File Sizes (approximate):**
- Installer: ~150-200 MB (includes WebView2 bootstrapper)
- MSI: ~120-150 MB
- Updater archive: ~80-100 MB (no WebView2)

### Build Configuration

The build is configured via:

<CodeGroup>
```json tauri.conf.json
{
  "build": {
    "beforeDevCommand": "npm run dev",
    "devUrl": "http://localhost:1420",
    "beforeBuildCommand": "npm run build",
    "frontendDist": "../dist"
  },
  "bundle": {
    "active": true,
    "createUpdaterArtifacts": true,
    "targets": "all",
    "resources": [
      "binaries/yt-dlp.exe",
      "binaries/spotdl.exe",
      "binaries/ffmpeg.exe",
      "binaries/ffprobe.exe"
    ]
  }
}
```

```toml Cargo.toml
[profile.release]
panic = "abort"        # Smaller binary, no unwinding
codegen-units = 1      # Better optimization, slower compile
lto = true             # Link-time optimization
opt-level = "s"        # Optimize for size
strip = true           # Remove debug symbols
```
</CodeGroup>

## Troubleshooting

### Issue: Rust compilation fails

**Error:**
```
error: linker `link.exe` not found
```

**Solution:**
Install Visual Studio C++ Build Tools:
```bash
winget install Microsoft.VisualStudio.2022.BuildTools
```

### Issue: Binary download script fails

**Error:**
```
Invoke-WebRequest : The request was aborted: Could not create SSL/TLS secure channel.
```

**Solution:**
Update PowerShell TLS settings:
```powershell
[Net.ServicePointManager]::SecurityProtocol = [Net.SecurityProtocolType]::Tls12
```

### Issue: WebView2 not found

**Error:**
```
Error: WebView2 runtime not found
```

**Solution:**
Install WebView2 Runtime:
```bash
winget install Microsoft.EdgeWebView2Runtime
```

### Issue: Frontend build fails with TypeScript errors

**Solution:**
```bash
# Clear node_modules and reinstall
rm -rf node_modules package-lock.json
npm install

# Run type check
npx tsc --noEmit
```

### Issue: Dev server port already in use

**Error:**
```
Error: Port 1420 is already in use
```

**Solution:**
```bash
# Kill the process using port 1420
# Windows:
netstat -ano | findstr :1420
taskkill /PID <PID> /F

# Linux/macOS:
lsof -ti:1420 | xargs kill -9
```

## Advanced Build Options

### Custom Build Targets

```bash
# Build specific bundle format
npm run tauri build -- --bundles nsis

# Build with debug symbols
npm run tauri build -- --debug

# Build for specific architecture
rustup target add x86_64-pc-windows-msvc
cargo build --release --target x86_64-pc-windows-msvc
```

### Environment Variables

```bash
# Custom updater endpoint
export TAURI_UPDATER_ENDPOINT=https://your-server.com/updates

# Enable debug logging
export RUST_LOG=debug

# Custom build number
export TAURI_BUILD_NUMBER=123
```

### Binary Signing (for releases)

Generate signing keys for the auto-updater:

```bash
npm run tauri signer generate -- -w ~/.tauri/ownstash.key
```

This creates:
- Private key: `~/.tauri/ownstash.key`
- Public key: Printed to console (add to `tauri.conf.json`)

<Warning>
  **Keep your private key secure!** Anyone with access can create fake updates.
</Warning>

## Development Tips

### Hot Reload Configuration

Tauri dev server automatically reloads on:
- ✅ Frontend file changes (instant HMR)
- ✅ Rust source changes (recompiles + restarts)
- ❌ `Cargo.toml` changes (requires manual restart)
- ❌ `tauri.conf.json` changes (requires manual restart)

### Debugging Rust Code

**VS Code** with `rust-analyzer`:

```json .vscode/launch.json
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "lldb",
      "request": "launch",
      "name": "Debug Tauri App",
      "cargo": {
        "args": ["build", "--manifest-path=./src-tauri/Cargo.toml"]
      },
      "cwd": "${workspaceFolder}"
    }
  ]
}
```

### Frontend DevTools

Open Chrome DevTools in the running app:

```bash
# Enable DevTools in dev mode (already enabled)
# Right-click in app → Inspect Element

# Or programmatically:
window.__TAURI__.invoke('open_devtools')
```
